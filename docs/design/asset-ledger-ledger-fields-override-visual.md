# 台账字段覆盖语义颜色与列简化

版本：v1.0  
日期：2026-02-06

## 背景

资产详情页的“台账字段”区域此前包含：

- 来源值
- 覆盖值
- 生效值

并且覆盖语义颜色仅在“盘点摘要”中可见，台账字段行缺少同等视觉语义，导致来源/覆盖关系不直观。

## 设计决策

1. 台账字段应用与盘点摘要一致的覆盖语义颜色方案（边框色）：
   - 未覆盖：灰色
   - 覆盖且与来源不同：红色
   - 覆盖且与来源相同：绿色
   - 覆盖但来源为空：蓝色
2. 移除“生效值”列，仅保留“字段 / 来源值 / 覆盖值”。
   - 语义上“生效值”与覆盖逻辑重复，易造成信息冗余。
3. 语义边框统一放在“覆盖值”列：
   - 盘点摘要与台账字段两块区域都遵循同一位置策略；
   - “采集值”列保持纯文本，避免列语义混淆。
4. 台账字段行高对齐常规表格：
   - 去掉覆盖值单元的额外最小高度设定，避免视觉上明显偏高。

## 实现要点

- 复用 `getOverrideVisualMeta` 计算每行语义颜色；
- 对“覆盖值”展示容器加 `border-l-2` 语义边框；
- 编辑态保持输入框，不叠加语义边框，避免干扰录入。
